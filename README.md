# manaakhz-docs — نظام إدارة المشروع مع الـ AI

## كيف يعمل النظام

```
_ai/
  CONTEXT.md    ← يُعطى للـ AI في بداية كل جلسة (صغير = tokens أقل)
  TODO.md       ← الـ AI يقرأه ويحدّثه بعد كل مهمة
  MOCK_DATA.md  ← يُقرأ فقط عند الحاجة لبيانات وهمية

docs/           ← الـ AI يكتب هنا توثيق كل صفحة بعد إنجازها
  layout.md | home.md | about.md | design.md | blog.md | contact.md
```

---

## الـ Prompt الجاهز لكل جلسة عمل

انسخ هذا وأرسله للـ AI في بداية كل محادثة:

```
اقرأ الملفين التاليين:
1. _ai/CONTEXT.md
2. _ai/TODO.md

ثم نفّذ المهمة الأولى غير المكتملة في TODO.
بعد الانتهاء:
- حدّث TODO.md (ضع ✅ على المهمة وأضفها لقسم "منجز")
- أنشئ أو حدّث ملف docs المناسب
```

---

## المرحلتان الأساسيتان

### المرحلة الأولى: بيانات وهمية

- كل الصفحات تعمل بـ `mock-data.ts`
- الواجهة كاملة ومرئية بدون Strapi
- سهل الاختبار والتعديل البصري

### المرحلة الثانية: ربط Strapi

- فقط `strapi.ts` يستبدل `mock-data.ts`
- الـ types والمكونات لا تتغير
- ربط صفحة واحدة في كل مرة

---

## قاعدة توفير الـ Tokens

| ما يقرأه الـ AI     | متى                                         |
| ------------------- | ------------------------------------------- |
| `CONTEXT.md`        | دائماً (في البداية)                         |
| `TODO.md`           | دائماً (في البداية)                         |
| `MOCK_DATA.md`      | فقط عند إنشاء mock-data.ts                  |
| `docs/X.md`         | فقط عند العمل على صفحة X                    |
| خطة المشروع الكاملة | **أبداً** — المعلومات مُلخّصة في CONTEXT.md |
